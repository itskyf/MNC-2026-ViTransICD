# DC-4: Weak Supervision Aggregation

## Objective

Build a deterministic silver-stage weak supervision aggregator that converts DC-3 candidate links into positive 3-character ICD weak labels with traceable evidence spans.

The task must reduce retrieval noise by prioritizing mention-grounded support over document-only retrieval support.

## Dependencies

This task depends on:

* DC-3 output:
  * `data/silver/<dataset_name>/candidate_links/<split>.jsonl`
* DC-2 or DC-2b output:
  * `data/silver/<dataset_name>/canonical_mentions/<split>.jsonl`
* DC-1 output:
  * `data/silver/<dataset_name>/documents/<split>.jsonl`
* ON-3 output:
  * `data/silver/icd10_official_pdf/coding_rules/rule_records.jsonl`

Do not depend on BM-1 or BM-2 outputs.

Do not re-run upstream tasks inside DC-4.

## Scope

Implement only deterministic aggregation from candidate links to positive weak labels.

Persist one row per `(doc_id, code_3char)` positive weak label.

Attach evidence spans from mention-grounded support only.

Use ON-3 per-code rules only as score adjustments and pruning signals.

Use ON-3 global rules only for validation and provenance checks.

Write outputs only under:

* `data/silver/<dataset_name>/weak_labels/`

## Out of Scope

Do not train a model.

Do not create negative labels.

Do not generate chapter-level or 4-character labels.

Do not implement a full ICD coding inference engine.

Do not infer principal diagnosis, mortality coding, or symptom fallback semantics from raw text.

Do not create any storage layer outside `data/{bronze,silver,gold}/...`.

## Required Data Structure

Create a new schema file:

* `src/mnc/schemas/weak_label.py`

Define two persisted schema types:

```python
from datetime import datetime
from typing import Literal

from pydantic import BaseModel, ConfigDict


class WeakEvidenceSpan(BaseModel):
    model_config = ConfigDict(extra="forbid")

    mention_id: str
    char_start: int
    char_end: int
    text: str
    methods: list[Literal["exact", "normalized", "fuzzy"]]
    score: float


class WeakLabelRecord(BaseModel):
    model_config = ConfigDict(extra="forbid")

    doc_id: str
    code_3char: str
    label_type: Literal["positive"]
    confidence: float
    rank: int
    support_methods: list[Literal["exact", "normalized", "fuzzy", "tfidf", "bm25"]]
    support_rule_ids: list[str]
    evidence_spans: list[WeakEvidenceSpan]
    created_at: datetime
```

Notes:

* `evidence_spans` must be non-empty.
* `support_methods` must be unique and sorted deterministically.
* `support_rule_ids` must be unique and sorted deterministically.

## Aggregation Unit

Aggregate candidates by:

* `(doc_id, code_3char)`

Within each group, collect:

* best score per method
* list of mention-backed candidate rows
* list of document-level retrieval rows
* applicable ON-3 rule rows for the same `code_3char`

## Scoring Policy

Use this deterministic method weight table:

* `exact`: `1.00`
* `normalized`: `0.90`
* `fuzzy`: `0.60`
* `tfidf`: `0.45`
* `bm25`: `0.50`

Compute:

* `mention_score = max(weight(method) * score)` over mention-backed candidates from `exact`, `normalized`, `fuzzy`
* `doc_score = max(weight(method) * score)` over document-level candidates from `tfidf`, `bm25`
* `rule_bonus = 0.10` if the code has at least one `include_note`
* `rule_penalty = 0.25` if the code has at least one `exclude_note` and `mention_score < 0.90`
* `rule_penalty += 0.10` if the code has at least one `code_first` or `use_additional_code` rule and `mention_score == 0.0`

Final confidence:

* `confidence = min(1.0, mention_score + 0.35 * doc_score + rule_bonus - rule_penalty)`

Round confidence to 6 decimal places before persistence.

## Emission Rules

Emit a positive weak label only when all conditions hold:

* `mention_score >= 0.60`
* `confidence >= 0.70`
* there is at least one mention-backed candidate row for the code
* the final code is a valid 3-character ICD code matching `^[A-Z][0-9]{2}$`

Do not emit document-only retrieval codes.

Do not emit more than 5 positive labels per document.

If more than 5 labels qualify, keep the top 5 by:

* `confidence` descending
* `mention_score` descending
* `code_3char` ascending

## Evidence Policy

Build evidence spans only from mention-backed candidates.

Join `CandidateLink.mention_id` to `MentionRecord`.

For each emitted `(doc_id, code_3char)` label:

* deduplicate evidence by `(mention_id, char_start, char_end)`
* merge methods for the same mention span
* set span score to the maximum weighted mention contribution for that span
* prefer `MentionRecord.normalized_text` as `text`; fall back to `MentionRecord.text`
* keep at most 3 evidence spans per label

Sort evidence spans by:

* `score` descending
* `char_start` ascending
* `mention_id` ascending

## Rule Usage

Use ON-3 code-scoped rules only for deterministic score adjustments.

Use these topics:

* `include_note`
* `exclude_note`
* `use_additional_code`
* `code_first`

Do not apply `general_note` in scoring.

Do not attempt semantic matching between note text and document text in DC-4.

Use ON-3 global rules only for:

* required input validation
* provenance that DC-4 consumed the official 3-character policy context

Do not attempt to derive diagnosis type from global rules in this task.

## Validation

Validate every output row with `WeakLabelRecord`.

Minimum checks:

* all emitted rows have `label_type="positive"`
* all `confidence` values are in `[0.0, 1.0]`
* all `rank` values start at 1 within each document and are contiguous
* all emitted rows have non-empty `support_methods`
* all emitted rows have non-empty `evidence_spans`
* all `evidence_spans` point to valid `MentionRecord` offsets
* all `code_3char` values are 3-character codes
* all output ordering is deterministic

Deterministic output ordering:

* sort rows by `doc_id`, then `rank`, then `code_3char`
